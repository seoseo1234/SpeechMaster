# 제품 요구사항 정의서 (PRD): AI SpeechMaster Pro (스피치마스터 프로)

**실시간 멀티모달(음성+비주얼+주파수) 통합 초등 발표력 진단 및 AI 처방 대시보드**

---

## 1. 프로젝트 개요 (Project Overview)

### 1.1 배경 및 목적
* **교육적 현안:** 초등학교 교실에서 저학년의 '한글 해득 부진 및 읽기 유창성 격차'와 고학년의 '발표 불안 및 표현력 부족'은 개별화 지도가 가장 시급한 영역입니다.
* **해결 방안:** 교사가 일일이 기록하기 어려운 학생들의 발표와 낭독 상태를 **음성(발음/성량), 시각(시선/태도), 언어(습관어/속도)** 데이터로 다차원 측정하여 분석합니다.
* **목표:** 초등 교직 전문성과 첨단 멀티모달 기술을 융합하여, 객관적인 진단 지표와 나이스(NEIS) 입력용 관찰평가 피드백을 자동으로 생성하는 완성도 높은 웹 애플리케이션을 구축합니다.

### 1.2 대상 사용자
* **초등 저학년 학생:** 한글 해득 부진, 조음(발음) 장애 위험이 있어 정밀한 낭독 교정이 필요한 학생
* **초등 고학년 학생:** 발표 불안이 있고 시선 처리, 발표 성량 및 말하기 속도 제어 훈련이 필요한 학생
* **지도 교사:** 학생별 다차원 스피치 데이터를 누적 관리하고 생활기록부/수행평가용 서술형 피드백을 추천받고자 하는 교사

## 2. 핵심 아키텍처 및 연동 기술 (Technical Architecture)

```
[클라이언트 브라우저 (HTML5 / Tailwind CSS / Vanilla JS 또는 React)]
  ├─ 🎤 Web Audio API ──────────> 실시간 데시벨(dB) 변동 폭 측정 및 주파수 시각화
  ├─ 📷 MediaPipe Face Mesh ────> 시선 이탈률 및 발표 자세 감지 (웹캠 연동)
  └─ 🎙️ Azure Speech SDK ───────> 한글 음소 레벨 발음 정확도(Accuracy) 및 유창성 진단
        │
        ▼ (통합 데이터 포맷 변환)
[Firebase Firestore] ───────────> 실시간 세션 로그 및 학생별 히스토리 누적 저장
        │
        ▼ (교사용 생성 트리거)
[OpenAI / Gemini API] ──────────> 정량 데이터 분석 및 나이스(NEIS)용 맞춤형 관찰평가 생성
```

## 3. 사용자 시나리오 및 핵심 플로우 (User Flow)

### 3.1 진입 및 사용자 로그인 (공통)
* **로그인:** 교사가 구글 계정(Firebase Auth)으로 로그인한 뒤, 학급 관리 대시보드에 진입합니다.
* **학생 선택:** 학생이 자신의 이름을 선택하고 오늘 연습할 모드를 선택합니다.
  * **[트랙 A] 초등 저학년 (한글 해득 & 발음 훈련 모드)**
  * **[트랙 B] 초등 고학년 (발성 & 스피치 프레젠테이션 모드)**

---

### 3.2 [트랙 A] 초등 저학년 모드 (발음 & 유창성 중심)

#### 1) 대기 및 환경 설정
* 화면에 아기자기한 마스코트 캐릭터와 함께 읽어야 할 오늘의 동화책 지문(또는 국어 교과서 문장)이 큰 글씨로 나타납니다.
* 마이크 테스트 버튼을 눌러 마이크 작동 여부를 확인합니다.

#### 2) 실시간 낭독 및 녹음 단계
* '시작' 버튼을 누르면 녹음이 시작됩니다.
* 학생이 지문을 읽는 동안 **Azure Cognitive Speech API(Pronunciation Assessment)**가 학생의 목소리를 실시간으로 추적하여 오리지널 지문 텍스트와 음소를 대조합니다.

#### 3) 실시간 분석 및 피드백 화면
* **정밀 발음 진단:** 정확히 읽은 단어는 초록색, 발음이 뭉개지거나 잘못 읽은 단어(예: 닭이 -> [다기]가 아닌 [다기/닥이] 등으로 오독)는 빨간색 하이라이트로 표시됩니다.
* **유창성 맵핑:** 문장 중간에 부자연스럽게 쉬어 간 구간(Pause)과 끊어 읽기 타이밍을 계산해 리듬감 점수를 매겨줍니다.
* **게이미피케이션:** "우와, 민수! 오늘 어려운 겹받침 글자 '닭'을 완벽하게 마스터했어! 발음 점수 95점!"과 같은 캐릭터 기반의 직관적인 피드백 카드가 나타납니다.

---

### 3.3 [트랙 B] 초등 고학년 모드 (발성, 유창성, 소리 크기 & 시선 중심)

#### 1) 발표 대본 준비 단계
* 학생이 오늘 수업 시간에 발표할 국어/사회 대본을 직접 입력하거나 교사가 미리 등록해 둔 예시 대본을 불러옵니다.
* 웹캠과 마이크 사용 권한을 수락하고 카메라 격자선 안으로 얼굴을 맞춥니다.

#### 2) 실시간 발표 및 분석 단계
* **실시간 성량 분석 (Web Audio API):** 발표 도중 데시벨(dB)의 미세한 크기 변화를 수집합니다. 소리가 너무 작으면 "조금 더 크게 말해볼까요?"라는 부드러운 가이드 배너가 화면 하단에 살짝 팝업됩니다.
* **시선 이탈 및 태도 트래커 (MediaPipe):** 웹캠 영상을 통해 실시간으로 얼굴의 각도와 눈동자의 좌표를 측정합니다. 발표 대본(바닥)만 너무 오래 쳐다보거나 고개를 과도하게 흔들면 '시선 흐트러짐'으로 감지해 데이터를 기록합니다.
* **습관어 실시간 감지:** 음성을 받아쓰면서 불필요한 간투사("어..", "음..", "이제", "막")의 사용 횟수를 누적 카운트합니다.

#### 3) 대시보드 피드백 화면
* 발표가 끝나면 **성량 분산 그래프, 시선 처리 안정도(%), 습관어 빈도 차트**가 종합 결과 보고서 형식으로 스크롤 가능한 대시보드에 고급스럽게 연동됩니다.

---

### 3.4 교사 전용 관리 대시보드 (Teacher Dashboard)

* **학급 스피치 현황판:** 학급 학생들의 평균 정확도 추이와 오늘 발표를 끝마친 학생 목록이 실시간으로 동기화됩니다.
* **성장 히스토리:** 특정 학생을 선택하면 학기 초부터 현재까지의 읽기 속도(WPM), 발음 정확도 향상 곡선이 차트(Chart.js)로 제공됩니다.
* **나이스(NEIS) 관찰평가 빌더 (AI Assistant):**
  * 교사가 '생활기록부 문구 생성' 버튼을 누르면 **OpenAI GPT-4o / Gemini 1.5 Pro**가 해당 학생의 누적 멀티모달 기록 데이터(성량, 속도, 시선 처리, 발음 데이터)를 분석합니다.
  * 지침에 따라 공손하고 전문적인 서술형 생활기록부 평가 피드백 문장을 자동으로 완성해 제공하며, 교사는 이를 바로 복사(Copy)해서 나이스에 활용할 수 있습니다.

## 4. 상세 기능 요구사항 (Functional Requirements)

### 4.1 시스템 및 인프라 (System & Infrastructure)
* **F-01 [인증]:** 구글 로그인 기능 연동 (교사 계정용). 학생은 로그인 없이 교사가 생성한 학급 코드를 통해 간이 등록 후 사용 가능.
* **F-02 [실시간 DB]:** Firebase Firestore를 백엔드로 활용하여 데이터 생성, 조회, 수정 및 교사 화면과 학생 화면 간의 실시간 데이터 동기화 구현.
* **F-03 [스토리지]:** 학생이 발표한 최종 음성 녹음 파일(.mp3 또는 .wav)을 Firebase Storage에 안전하게 저장하여 교사가 추후 재생하여 들을 수 있도록 구현.

### 4.2 저학년 읽기/발음 모듈 (Lower-Grade Module)
* **F-04 [음소 정밀 분석]:** Microsoft Azure Speech SDK의 `PronunciationAssessmentConfig`를 활성화하여 음성 신호를 분석하고 글자/음소 단위의 정확도(Accuracy), 유창성(Fluency), 완결성(Completeness) 점수 리턴.
* **F-05 [텍스트 하이라이팅]:** 원래 텍스트와 Azure 분석 결과를 비교하여 오독한 글자(Misspeaking), 생략한 글자(Omission)를 판단하고 CSS로 색상을 다르게 적용(정상: 초록, 오류: 빨강).
* **F-06 [한글 자모 분해 피드백]:** 한글 자모 분리 라이브러리(`Hangul.js`)를 연동하여 특정 초/중/종성 중 반복적으로 틀리는 소릿값을 교사 대시보드에 요약 노출.

### 4.3 고학년 스피치 모듈 (Upper-Grade Module)
* **F-07 [실시간 데시벨 감지]:** Web Audio API의 `AnalyserNode`를 이용해 실시간 마이크 입력 값에서 RMS(Root Mean Square)를 구한 뒤 데시벨(dB) 값으로 변환하여 실시간 오디오 파형(Waveform) 시각화.
* **F-08 [발표 시선/태도 측정]:** MediaPipe Face Mesh 또는 Face Landmarker를 사용하여 카메라 입력에서 정면 벡터 기준 눈동자 좌표 및 고개의 롤/피치 각도를 감지, 전체 녹화 시간 중 정면 주시 비율(%)을 계산하여 리턴.
* **F-09 [습관어 및 속도 분석]:** STT가 처리한 텍스트에서 불필요한 단어들(`어`, `음`, `그니까`, `이제`, `막`)을 정규표현식(Regex)으로 검색하여 횟수를 집계하고 전체 발화 시간 대비 분당 단어 수(WPM) 계산.

### 4.4 AI 및 대시보드 모듈 (AI & Teacher Dashboard Module)
* **F-10 [스피치 데이터 시각화]:** Chart.js를 사용해 개인별 스피치 변화 곡선(꺾은선 그래프) 및 다차원 역량 평가(성량 안정성, 유창성, 시선 처리, 발음 정확도)를 방사형 차트(Radar Chart)로 표현.
* **F-11 [AI 생활기록부 평가 작성]:** OpenAI/Gemini API를 백그라운드에서 호출하되, 시스템 프롬프트(System Prompt)에 초등 교육과정 교과 성취기준 및 나이스(NEIS) 기록 서식을 학습시켜 정제된 어조의 관찰 의견 텍스트 생성.

## 5. 비기능 요구사항 (Non-Functional Requirements)

* **보안 및 규정 준수 (Security & Privacy):**
  * 학생들의 실시간 웹캠 및 음성 데이터는 브라우저 단에서 실시간 분석 후 파기하며, 데이터베이스에 저장되는 정보는 비식별화된 학생 번호와 요약 데이터(숫자 점수 및 텍스트 로그)로 한정하여 개인정보 유출 리스크를 원천 차단합니다.
  * 유료 API 키(Azure, OpenAI, Gemini)는 클라이언트 코드에 하드코딩하지 않고, Firebase Cloud Functions(서버리스 함수)를 사용하거나 사용자가 설정 페이지에서 본인의 API Key를 입력받아 브라우저 보안 스토리지(LocalStorage)에 암호화하여 저장 후 사용하도록 구현합니다.
* **사용성 및 반응형 디자인 (UX/UI):**
  * 저학년: 시인성이 높은 큰 아이콘, 둥글둥글한 라운드 처리, 편안함을 주는 파스텔 Yellow & Green UI 테마 적용.
  * 고학년 및 교사: 정보의 가독성이 높고 신뢰감을 주는 모던한 Slate Navy & White 대시보드 UI 테마 적용.

## 7. 심사위원단 대상 포트폴리오 차별화 포인트 (Portfolio Winning Strategy)

1. **에듀테크와 교직 전문성의 환상적인 융합**
   * 일반 개발자들의 발표 보조 프로그램은 단순히 음성 인식을 받아 적는 데 그치지만, 본 서비스는 **"초등 저학년 한글 해득 장애 예방"**과 **"고학년 수행평가 및 발표 역량 강화"**라는 실질적인 학교 현장의 교과 성취 기준을 타겟팅하여 프로그램의 목적성을 고도화했습니다.
2. **기술적 고난도 아키텍처 증명**
   * 클라이언트 웹 브라우저 단에서 인공지능 모델인 **MediaPipe Face Mesh**를 가볍게 구동하여 시선을 트래킹하고, **Web Audio API**를 사용해 오디오 노드를 분석하는 등 고성능 프론트엔드 역량을 보여줍니다.
   * 백엔드에서는 **Azure AI Speech의 정밀 음소 진단과 OpenAI/Gemini LLM 비서**를 매끄럽게 파이프라인으로 연결하여 완성도 높은 풀스택 서버리스 시스템을 구축했음을 증명합니다.
3. **교사 행정 업무 경감(업무 혁신)**
